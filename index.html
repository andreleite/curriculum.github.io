Vérios Investimentos (www.comparacaodefundos.com) de jan/2011 a ago/2013

Fui contratado pela Vérios Investimentos para construir um sistema que fosse capaz de conversar com uma API SOAP da CVM (Comissão de Valores Mobiliários) e permitir o consumo dos dados de rentabilidade, patrimônio, número de cotistas etc. de todos os 15 mil fundos de investimento do Brasil.

A quantidade de dados é enorme (para cada dia útil desde 1996, temos valores de cota, patrimônio, aplicação, resgate e cotistas de 15 mil fundos). Para dar conta de processar essa massa de dados, optei pelo uso do banco de dados nosql Redis, que tem uma velocidade imbatível porque mantém os dados na memória. Tudo isso orquestrado por uma aplicação que usa node.js e express num esquema REST.

No frontend, a necessidade de mostrar os dados de modo palatável me fez usar ferramentas como flot (uma library que produz gráficos usando canvas), e d3.js (que usa svg).Também usei Require.js (para manter o código modular), Backbone.js (ajuda muito a organizar as coisas) e Bootstrap (para podermos desenvolver novas funcionalidades mais rapidamente.)

A aplicação consome os dados no formato JSON da aplicação node, e faz 90% do processamento no próprio browser do usário. Desde a criação dos gráficos até cálculos como volatilidade e risco, acontecem client-side.

Com uma massa de usuários que em agosto de 2013 chegou a 25 mil visitantes requisitando mais de 240 mil pageviews, os custos para manter o sistema nunca ultrapassaram os US$ 100/mensais.

Observação: As restrições de custo que a empresa tinha para colocar o projeto de pé fizeram com que eu ficasse responsável por toda a identidade visual do projeto.
